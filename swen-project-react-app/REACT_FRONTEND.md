# Trail Planner - React Frontend

This is the React frontend application for the Trail Planner system, built with Vite and React.

## Overview

The Trail Planner frontend provides:
- **Dashboard**: Interactive graphs and data visualization for trail usage
- **Trail Management**: Add, edit, and organize trails and areas
- **Device Management**: Associate devices with trails
- **List View**: Overview of all trails with weekly counts, battery status, and last updated dates
- **Graph View**: Time-series visualization of trail usage data
- **Authentication**: Cognito-based user authentication

## Features

### Dashboard Views

1. **Graph View** (default):
   - Interactive Plotly.js graphs
   - Date range filtering
   - Granularity selection (Hourly, Daily, Weekly, Monthly, Yearly)
   - Multi-trail selection
   - Areas filtering

2. **List View**:
   - Table of all trails
   - Weekly count (entries from last 7 days)
   - Battery status (color-coded)
   - Last updated date (MM/DD/YYYY format)

### Trail Management

- **Add Trail**: Create new trails
- **Edit Trail**: Update trail names and area associations
- **Delete Trail**: Remove trails and associated data
- **Areas**: Organize trails into wilderness areas

### Device Management

- **Associate Device**: Link devices to specific trails
- **Device Metadata**: View battery status and last update times

## Prerequisites

- **Node.js** 16.x or higher
- **npm** or **yarn**
- **AWS Cognito** configured (credentials injected via Terraform)
- **AWS dev-role** configured by following [AWS User Creation Guide](../docs/AWS-User-Creation-Guide.md) then [AWS Configure Role Guide](../docs/AWS-Configure-Role-Guide.md)

## Installation

1. Install dependencies:

```bash
npm install
```

## Setup Env

There is a file [setup_env.py](setup_env.py) that when the **AWS dev-role** prerequisite is complete will automatically be run when the development server is started. This code creates a .env file and fills it with the neccisary information to link the development server to the tst backend.

## Development

Start the development server:

```bash
npm run dev
```

> **Note**: The website is heavily intertwined with AWS so you will likely encounter site-breaking issues running the development server.

## Deployment

The React app is automatically deployed to S3 via Terraform. See the main [README.md](../README.md) and [DEPLOYMENT_CLOUDFRONT_ROUTE53.md](../terraform/DEPLOYMENT_CLOUDFRONT_ROUTE53.md) for deployment instructions.

### Environment Variables

The application uses environment variables configured in `.env` or `vite.config.ts`, which is automatically linked to the existing tst deployment when running `npm run dev`:

- `VITE_API_URL`: API Gateway base URL (set by Terraform)
- `VITE_COGNITO_REGION`: AWS region for Cognito (defaults to `us-east-1`)
- `VITE_COGNITO_USER_POOL_ID`: Cognito User Pool ID (set by Terraform)
- `VITE_COGNITO_CLIENT_ID`: Cognito User Pool Client ID (set by Terraform)

### Cognito Configuration

Cognito configuration is automatically generated by Terraform and placed in:
- `src/cognito/config.json`

This file contains:
- `region`: AWS region
- `userPoolId`: Cognito User Pool ID
- `clientId`: Cognito User Pool Client ID

## Key Components

### Dashboard (`dashboard.tsx`)

Main application component that:
- Manages state for trails, areas, and graph data
- Handles view switching (graph/list)
- Coordinates data fetching and display
- Manages modal states

### TrailSelector (`components/trailselector.tsx`)

React-Select based component for:
- Selecting wilderness areas
- Selecting individual trails
- Multi-select support
- "All Trails" option

### EditTrailModal (`components/EditTrailModal.tsx`)

Modal for:
- Creating new trails
- Editing existing trail information
- Assigning trails to areas
- Deleting trails

### EditAreaModal (`components/EditAreaModal.tsx`)

Modal for:
- Creating new areas
- Editing area names
- Managing trail-to-area associations
- Ensuring trails are only in one area

## Troubleshooting

### Build Errors

- Ensure all dependencies are installed: `npm install`
- Check Node.js version: `node --version` (should be 16+)
- Clear cache and rebuild: `rm -rf node_modules dist && npm install && npm run build`

### Authentication Issues

- Verify environment variables are set correctly:
  - `VITE_COGNITO_USER_POOL_ID`
  - `VITE_COGNITO_CLIENT_ID`
  - `VITE_COGNITO_REGION`
- Check Cognito User Pool is created in AWS
- Verify API Gateway URL is correct in environment variables

### API Connection Issues

- Verify `VITE_API_URL` is set correctly
- Check API Gateway is deployed and accessible
- Verify Cognito token is valid (check expiration)
